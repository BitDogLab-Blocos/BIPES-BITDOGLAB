**Tarefa:** Corrigir os menus de seleção de pinos digitais para a Raspberry Pi Pico.

**Contexto:**
O prompt anterior corrigiu com sucesso o bloco "ler entrada analógica", que agora exibe os pinos ADC corretos. No entanto, os blocos de I/O digital e PWM (`ajustar pino de saida`, `ler entrada digital`, `pino`, `PWM`) ainda não mostram a lista de pinos disponíveis, exibindo "não definido".

**Análise do Problema:**
O sistema provavelmente utiliza uma estrutura de dados central (um objeto JavaScript) para definir as pinagens de cada placa suportada. Os blocos de I/O digital devem estar buscando a lista de pinos dessa estrutura central com base na placa selecionada ("Raspberry Pi Pico"). A correção, portanto, não é em cada bloco individualmente, mas sim na fonte de dados que eles consultam.

**Instruções para você, IA:**

1.  **Localize a Configuração da Placa:** Procure no código por uma variável ou objeto que armazene as configurações de todas as placas, algo como `devices` ou `board_configurations`.

2.  **Encontre a Entrada da Raspberry Pi Pico:** Dentro desse objeto de configurações, localize a entrada para a Raspberry Pi Pico. A chave pode ser `"RPI_Pico"`, `"Raspberry Pi Pico"` ou algo similar.

3.  **Adicione ou Modifique a Lista de Pinos (`pinout`):**
    * Dentro do objeto da Raspberry Pi Pico, adicione uma propriedade chamada `pinout`.
    * O valor desta propriedade deve ser um *array de arrays*, contendo a lista completa de todos os pinos GPIO de uso geral, de `GP0` a `GP28`. O formato deve ser `[["NOME_EXIBIDO", "VALOR"], ...]`. Para este caso, ambos serão iguais (ex: `["GP0", "0"]` ou `["GP0", "GP0"]`).

**Exemplo de como a Estrutura de Dados deve Ficar:**
Procure por um código semelhante a este e adicione ou modifique a seção `RPI_Pico`.

```javascript
// O nome da variável pode ser diferente, como "UI.workspace.devices" ou "boardData"
var devices = {
    "ESP32": {
        "name": "ESP32",
        "pinout": [ /* ... pinos do ESP32 ... */ ]
    },

    // ... outras placas ...

    // ADICIONE OU MODIFIQUE ESTA SEÇÃO
    "RPI_Pico": {
        "name": "Raspberry Pi Pico",
        "doc": "devinfo/pico.html", // pode ter outras propriedades
        "pinout": [
            ["GP0", "0"], ["GP1", "1"], ["GP2", "2"], ["GP3", "3"],
            ["GP4", "4"], ["GP5", "5"], ["GP6", "6"], ["GP7", "7"],
            ["GP8", "8"], ["GP9", "9"], ["GP10", "10"], ["GP11", "11"],
            ["GP12", "12"], ["GP13", "13"], ["GP14", "14"], ["GP15", "15"],
            ["GP16", "16"], ["GP17", "17"], ["GP18", "18"], ["GP19", "19"],
            ["GP20", "20"], ["GP21", "21"], ["GP22", "22"],
            // Pinos de controle (RUN, 3V3, etc.) geralmente não são incluídos,
            // mas todos os GPIOs (0-22 e 26-28) sim.
            ["GP26", "26"], ["GP27", "27"], ["GP28", "28"]
        ]
    }
};
```

**Observação Importante:**
O bloco `ler entrada analógica` deve continuar usando sua lista específica e já corrigida de `["GP26", "GP27", "GP28"]`. Esta modificação na configuração central deve afetar apenas os blocos de uso geral (digital e PWM), garantindo que eles agora exibam a lista completa de pinos GPIO para seleção.
